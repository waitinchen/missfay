# 🌐 正确的访问地址说明

## 问题说明

### 为什么 `http://0.0.0.0:8000` 无法访问？

`0.0.0.0` 是**服务器绑定地址**，不是客户端访问地址。它表示服务器监听所有网络接口，但浏览器无法直接访问这个地址。

## 正确的访问地址

### ✅ 使用以下地址之一：

1. **http://localhost:8000** （推荐）
2. **http://127.0.0.1:8000**

这两个地址都指向本地计算机，可以正常访问。

## 访问步骤

### 步骤 1: 打开浏览器

### 步骤 2: 在地址栏输入

```
http://localhost:8000
```

或

```
http://127.0.0.1:8000
```

### 步骤 3: 按 Enter 键

应该能看到心菲的聊天界面。

## 验证服务是否运行

### 方法 1: 检查端口占用

```powershell
netstat -ano | findstr "8000"
```

如果看到 `0.0.0.0:8000` 或 `127.0.0.1:8000`，说明服务正在运行。

### 方法 2: 测试健康检查端点

```powershell
python -c "import requests; r = requests.get('http://localhost:8000/health'); print('OK' if r.ok else 'Failed')"
```

如果返回 `OK`，说明服务正常。

## 如果仍然无法访问

### 检查 1: 服务是否正在运行

查看 PowerShell 窗口，应该看到：
```
Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

### 检查 2: 防火墙设置

Windows 防火墙可能会阻止本地连接。如果无法访问，可以：
1. 暂时关闭防火墙测试
2. 或添加防火墙规则允许端口 8000

### 检查 3: 浏览器缓存

1. 按 `Ctrl+Shift+Delete` 清除浏览器缓存
2. 或使用无痕模式打开
